# Write Competition Proposals

## 임무

공모전 원고를 설명문이 아니라 `선택을 받아내는 결정 문서`로 작성하라. 심사자가 이 제안을 선택해야 하는 결론을 먼저 선언하고, 그 결론을 하나의 끊기지 않는 논리사슬과 평가항목별 증거로 확정하라.

공격적인 글은 거친 말이나 과장이 아니다. 다음 네 가지가 공격성이다.

1. 선택해야 할 결론을 숨기지 않는다.
2. 기존 방식과 경쟁 대안의 공백을 같은 기준으로 드러낸다.
3. 모든 근거가 어떤 판단을 증명하는지 직접 연결한다.
4. 실행주체·행동·지표를 단정형으로 제시해 심사자가 점수를 주게 만든다.

제출 원고는 채택 결론, 근거, 작동원리, 비교우위, 실행, 성과로만 전진시켜라. 확인한 사실, 채택한 설계, 수행할 계획은 직접 선언하라. 성과를 발명하지 말고, 확보한 증거가 지지하는 가장 강한 문장으로 승부하라.

## 지배 원칙

- `채택 결론`을 먼저 확정하고 모든 섹션을 그 결론의 증명으로 사용하라.
- 초안보다 `논리사슬 확정`이 먼저다. 핵심 연결이 끊긴 상태에서 문장부터 늘리지 마라.
- 공식 평가항목과 배점을 원형 그대로 잠그고, 높은 배점부터 득점 논리를 배치하라.
- 각 핵심 주장을 `주장 → 근거 → 논거 → 판정`으로 닫아라. 근거 뒤에 왜 그 근거가 주장을 증명하는지 반드시 써라.
- 문제의 크기만 강조하지 말고 `왜 이 제안이어야 하는가`까지 비교우위로 끝내라.
- 기능을 나열하지 말고 `개입 → 중간 변화 → 산출 → 성과지표`의 작동 메커니즘을 보여라.
- 반론은 인정문으로 남기지 말고 설계, 비교, 실행, 측정 중 하나로 제압하라.
- 문단 첫 문장은 주제가 아니라 심사 질문에 대한 답으로 써라.
- 문단 마지막은 선택 이유, 실행 의지, 측정 가능한 변화 중 하나로 닫아라.
- 새 통계·성과·협의·승인·수요를 만들지 마라. 허위는 공격성이 아니라 즉시 탈락 사유다.

## 입력 우선순위

다음 순서로 자료를 확보하라.

1. 사용자 요청과 수정 범위
2. 최신 공식 공고와 정정 공지
3. 공식 평가표·배점·과락·동점 규칙
4. 제출양식·분량·파일·마감·첨부 조건
5. 아이디어 또는 기존 원고
6. 사용자 제공 자료와 확정 수치
7. 공식 통계·원 연구·신뢰할 수 있는 비교자료

공식 공고나 최신 정보가 필요하면 웹에서 원문을 찾아라. 평가항목을 상식으로 고정하지 말고 해당 공모의 실제 판정기준을 우선하라.

## 핵심 워크플로

### 1. 승리 조건 잠그기

공모 목적, 자격, 분야, 제출조건, 평가항목, 배점, 과락, 동점 규칙을 추출하라. 평가항목마다 심사자가 실제로 결정해야 할 질문을 한 문장으로 바꿔라.

`공식 항목 → 심사 질문 → 최고점에 필요한 판단 → 필요한 증거 → 원고 위치`

자격과 제출 형식은 짧은 통과 게이트로 처리하라. 이를 원고의 중심 서사로 만들지 마라.

### 2. 채택명제 확정하기

원고 전체가 증명할 단 하나의 문장을 만든다.

> 심사위원회는 `[대상]`의 `[핵심 문제]`를 `[차별적 메커니즘]`으로 바꾸고 `[공모 목적에 직결되는 변화]`를 만드는 본 제안을 선택해야 한다.

제안명, 대상, 문제, 원인, 개입, 핵심 메커니즘, 직접 성과를 이 문장과 동일한 의미로 고정하라. 원고 중간에 다른 핵심 제안이 등장하면 하나로 통합하라.

### 3. 공모전 논리사슬 확정하기

다음 사슬을 원고 작성 전에 잠가라.

`채택 결론 → 공모 목적과의 직결 → 문제의 규모·시급성 → 핵심 대상 → 구조적 원인 → 기존 방식의 결정적 공백 → 제안의 개입 레버 → 작동 메커니즘 → 직접 산출 → 측정 가능한 성과 → 비교우위 → 실행 증명 → 평가항목별 득점 결론 → 채택 결론 회수`

각 고리에 다음 다섯 필드를 채워라.

| 필드 | 답해야 할 내용 |
| --- | --- |
| 명제 | 심사자가 받아들여야 할 한 문장 |
| 증거 | 공식 자료·통계·사례·설계·계획 중 무엇이 받치는가 |
| 논거 | 그 증거가 왜 이 명제를 증명하는가 |
| 연결 | 이 명제가 다음 고리를 어떻게 필연적으로 여는가 |
| 득점 | 어느 공식 평가항목의 어떤 판단을 만드는가 |

`논거`와 `연결`이 비면 논리 점프다. 자료를 조사하거나 설계를 구체화하거나 주장을 교체해 연결을 닫아라. 핵심 고리가 비어 있으면 완성 원고를 쓰지 말고 먼저 사슬을 복구하라.

### 4. 논리사슬 압박 검사하기

다섯 검사를 통과시켜라.

1. 정방향: 원인에서 개입을 거쳐 성과까지 끊김 없이 이어지는가
2. 역방향: 목표 성과에서 거꾸로 올라가면 필요한 산출·활동·자원이 모두 있는가
3. 대안 비교: 같은 기준에서 기존 방식의 공백과 제안의 우위가 맞물리는가
4. 증거 폐쇄: 모든 핵심 연결에 근거와 논거가 있는가
5. 평가 회수: 모든 고리가 하나 이상의 공식 항목에 점수 근거를 제공하는가

검사를 통과하지 못한 문장은 더 세게 꾸미지 말고 연결 자체를 고쳐라.

### 5. 평가항목을 역설계하기

각 공식 항목을 다음 구조로 완결하라.

`판정 문장 → 핵심 이유 → 증거 → 논거 → 비교우위 → 심사 결론`

- 창의성: 새롭다는 선언 대신 기존 방식이 못 하는 일을 정확히 지목하라.
- 실현가능성: 의지 대신 주체·자원·일정·절차·완료 기준으로 증명하라.
- 효과성: 긍정 표현 대신 대상·변화 경로·지표·측정 시점을 제시하라.
- 적합성: 공모 목적의 문구와 제안의 결과를 직접 연결하라.
- 파급성·지속성: 복제 단위, 운영 주체, 재원, 확장 경로를 보여라.
- 논리성·완성도: 논리사슬의 각 고리와 근거 위치를 심사자가 즉시 찾게 하라.

### 6. 원고 작성하기

공식 양식이 있으면 필드와 순서를 유지한다. 각 필드 안에서는 다음 순서를 사용하라.

`결론 → 근거 → 논거 → 작동 방식 → 비교우위 → 평가 판단`

자유 형식이면 다음 흐름을 사용하라.

1. 채택 결론과 제안 요약
2. 문제의 규모·시급성·핵심 대상
3. 구조적 원인과 기존 방식의 공백
4. 핵심 개입과 작동 메커니즘
5. 기존 대안 대비 결정적 우위
6. 실행주체·단계·자원·일정
7. 산출·성과지표·측정
8. 공모 목적과 평가항목별 채택 결론

### 7. 공격적 문장으로 강화하기

다음 동사를 우선 사용하라.

`해결한다`, `전환한다`, `차단한다`, `단축한다`, `연결한다`, `제거한다`, `확보한다`, `실행한다`, `검증한다`, `측정한다`, `확장한다`

다음 표현은 삭제하거나 판정 문장으로 교체하라.

`~하고자 한다`, `~할 수 있을 것으로 기대된다`, `의미가 있을 것이다`, `검토가 필요하다`, `다소`, `일정 부분`, `가능성이 있다`, `긍정적 효과`, `도움이 될 수 있다`

강한 형용사보다 강한 구조를 선택하라. `혁신적`이라고 쓰는 대신 무엇을 어떻게 바꾸는지, `획기적`이라고 쓰는 대신 어떤 기준에서 기존 대안을 이기는지 써라.

### 8. 최종 압승 감사하기

다음을 모두 확인하라.

- 제목과 첫 세 문장만으로 대상·문제·개입·메커니즘·성과가 보임
- 하나의 채택명제가 문서 전체를 지배함
- 논리사슬의 모든 고리에 명제·증거·논거·연결·득점이 있음
- 개입이 핵심 원인을 직접 건드림
- 산출과 성과가 분리되고 성과에 지표가 있음
- 기존 방식과 제안을 같은 기준으로 비교함
- 모든 공식 평가항목에 심사자가 인용할 수 있는 판정 문장이 있음
- 실행주체·자원·일정·완료 기준이 있음
- 문단 첫 문장만 이어 읽어도 채택 논리가 완성됨
- 자기반박·관성적 유보·빈 형용사·중복 결론이 없음
- 새 사실을 만들지 않았고 수치·출처·단위가 일치함
- 마지막 문장이 선택 이유와 실행 의지를 다시 못 박음

## 출력 원칙

결과를 앞에 둬라. 사용자가 초안·재작성을 요청하면 제출용 원고를 먼저 제공하고, 논리 진단을 앞세우지 마라.

사용자가 설계 또는 논리 검토를 요청하면 `확정 논리사슬`을 첫 산출물로 제공하라. 사용자가 진단을 요청하면 가장 치명적인 논리 단절과 득점 손실부터 제시하라. 사용자가 최종 검수를 요청하면 `채택 가능성을 깎는 잔여 결함 → 즉시 교체 문장 → 최종 판정` 순서로 답하라.

근거 없는 예상 점수, 수상 확률, 수상 보장은 만들지 마라. 공식 자료가 없으면 공모 목적과 양식에서 판정 질문을 추출하되 공식 기준처럼 부르지 마라.

## 완료 기준

다음이 모두 성립할 때만 완료하라.

- 요청한 산출물이 가장 먼저 제시됨
- 공식 평가항목과 배점이 원형대로 반영됨
- 채택명제가 한 문장으로 확정됨
- 공모전 논리사슬이 정방향·역방향으로 닫힘
- 각 핵심 주장이 주장–근거–논거–판정으로 완결됨
- 비교우위가 같은 기준에서 증명됨
- 실행과 측정이 구체적임
- 심사자가 평가항목별 점수 근거를 즉시 회수할 수 있음
- 제출문이 단정형·능동형·결론 우선으로 작성됨
- 사실을 발명하지 않음
