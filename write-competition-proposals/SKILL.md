# Write Competition Proposals

## 임무

공모전 원고를 처음 읽는 심사자가 헤매지 않고 결론까지 따라갈 수 있는 결정 문서로 작성하라. 어려운 표현이나 과장으로 밀어붙이지 말고, 독자가 품는 질문에 정해진 순서로 답하라.

> 무엇을 제안하는가 → 무엇이 문제인가 → 왜 생겼는가 → 무엇으로 해결하는가 → 실제로 가능한가 → 무엇이 달라지는가 → 왜 선택해야 하는가

공식 평가항목은 이 흐름에 증거를 배치하는 기준으로 사용하라. 평가항목 때문에 글의 기본 방향을 바꾸거나 같은 결론을 여러 구조로 반복하지 마라.

## 유일한 표준

### 문서 구조

모든 원고는 다음 7부 구조만 사용하라.

`1. 제안 요약 → 2. 문제 상황 → 3. 문제의 원인 → 4. 해결 방안 → 5. 차별성과 실현 가능성 → 6. 기대 효과 → 7. 결론`

제목은 대상의 문제와 해결책이 드러나게 작성하라. 공식 양식이 있으면 필드명과 순서를 유지하되 각 필드의 역할을 이 7부 구조에 대응시켜라. 공식 양식에서도 새로운 글쓰기 순서를 만들지 마라.

### 문단 공식

모든 문단은 예외 없이 다음 공식 하나만 사용하라.

`주장 → 설명 → 근거·사례 → 소결론`

1. `주장`: 문단이 증명할 한 가지 판단을 첫 문장에 쓴다.
2. `설명`: 그 판단이 성립하는 이유·원인·작동 방식을 풀어 쓴다.
3. `근거·사례`: 확인된 통계·사례·비교·설계·실행계획으로 설명을 뒷받침한다.
4. `소결론`: 근거의 의미를 해석하고 해당 섹션의 판단 또는 다음 섹션으로 연결한다.

다른 문단 공식을 병용하지 마라. 근거를 제시한 뒤 해석을 생략하지 말고 반드시 소결론에서 주장과 연결하라.

## 지배 원칙

- 첫 문단에서 제안의 주제와 핵심 결론을 알게 하라.
- 한 문단에는 중심 주장 하나만 둬라.
- 현상, 문제, 원인을 구분하라.
- 해결 방안을 구조적·실질적 원인과 일대일로 연결하라.
- 해결 방안에는 실행주체·대상·행동·운영방식·기존 방식과의 차이를 포함하라.
- 실현 가능성은 주체·자원·절차·일정·위험 대응·완료 기준으로 증명하라.
- 기대 효과는 직접 효과·확장 효과·장기 효과로 나누고 측정 기준을 붙여라.
- 같은 대상을 같은 이름으로 부르고 핵심 수치·기간·주체를 끝까지 통일하라.
- 한 문장에는 핵심 하나만 두고 주어와 서술어를 가깝게 배치하라.
- 새 통계·성과·수요·협의·승인·예산·성능을 만들지 마라.

## 입력 우선순위

다음 순서로 자료를 확보하라.

1. 사용자 요청과 수정 범위
2. 최신 공식 공고와 정정 공지
3. 공식 평가표·배점·과락·동점 규칙
4. 제출양식·분량·파일·마감·첨부 조건
5. 아이디어 또는 기존 원고
6. 사용자 제공 자료와 확정 수치
7. 공식 통계·원 연구·신뢰할 수 있는 비교자료

공식 공고나 최신 정보가 필요하면 원문을 확인하라. 공식 평가항목이 없으면 공모 목적과 양식에서 보조 질문을 도출하되 공식 기준이라고 부르지 마라.

## 핵심 워크플로

### 1. 작업 모드와 승리 조건 확정

설계·초안·진단·재작성·최종 검수 중 작업 모드를 정하라. 공모 목적, 자격, 제출조건, 공식 평가항목, 배점, 과락, 동점 규칙을 추출하라.

평가항목마다 다음을 기록하라.

`공식 항목 → 심사 질문 → 필요한 판단 → 필요한 근거 → 7부 구조의 원고 위치`

### 2. 핵심 결론 확정

원고 전체가 주장할 한 문장을 확정하라.

> `[실행주체]`는 `[대상]`이 겪는 `[핵심 문제]`의 `[핵심 원인]`을 해결하기 위해 `[해결 방안]`을 실행해야 한다.

제안명, 대상, 문제, 원인, 해결 방안, 실행주체, 직접 효과를 이 결론과 같은 의미로 고정하라.

### 3. 문제–원인–해결 대응 확정

문제를 겪는 대상과 구체적 손실을 적고, 손실을 만드는 원인을 분리하라. 원인이 여러 개면 해결 방안도 같은 번호와 순서로 대응시켜라.

`문제 → 원인 1·2·3 → 해결 방안 1·2·3 → 실행 조건 → 직접 효과`

원인과 직접 연결되지 않는 기능은 삭제하거나 부가기능으로 낮춰라.

### 4. 7부 구조에 내용 배치

1. `제안 요약`: 현재 상황, 핵심 문제, 핵심 결론, 실행 방향을 한 문단 안에서 밝힌다.
2. `문제 상황`: 누가 어떤 어려움을 겪으며, 지속될 때 어떤 손실이 발생하는지 설명한다.
3. `문제의 원인`: 현상과 원인을 구분하고 문제를 반복시키는 구조와 기존 대응의 한계를 설명한다.
4. `해결 방안`: 누가 누구를 대상으로 무엇을 어떤 방식으로 실행하는지 제시한다.
5. `차별성과 실현 가능성`: 같은 기준에서 기존 방식과 비교하고 주체·자원·절차·일정·위험 대응·완료 기준을 제시한다.
6. `기대 효과`: 직접 효과, 확장 효과, 장기 효과와 각 효과의 측정 기준을 제시한다.
7. `결론`: 새로운 정보 없이 필요성·원인 적합성·실행 가능성·핵심 효과를 묶어 선택 이유를 확정한다.

### 5. 단일 문단 공식으로 작성

각 섹션을 `주장 → 설명 → 근거·사례 → 소결론` 문단으로 작성하라. 문단 첫 문장만 이어 읽어도 7부 구조가 자연스럽게 연결되어야 한다. 문단이 새 주장·설명·근거·사례·소결론을 추가하지 않으면 삭제하라.

### 6. 문장을 구체화

- 행위자를 문장 앞에 둬라.
- `활성화`, `강화`, `제고`, `혁신`, `연계` 뒤에는 무엇이 어떻게 달라지는지 써라.
- 수치 뒤에는 그 수치가 의미하는 문제·원인·설계 판단을 써라.
- 계획에는 담당·시점·절차·완료 기준을 붙여라.
- 확인한 사실, 확정한 설계, 실행 계획, 목표, 추정을 구분하라.
- 배경 일반론, 현황 반복, 근거 없는 최상급, 관성적 완곡어를 삭제하라.

### 7. 최종 검수

다음을 확인하라.

- 제목과 첫 문단에서 주제와 핵심 결론이 보임
- 7부 구조가 빠짐없이 같은 순서로 진행됨
- 모든 문단이 단일 문단 공식을 따름
- 문제와 원인이 구분됨
- 각 원인에 해결 방안이 직접 대응함
- 실행주체·대상·행동·운영방식이 구체적임
- 차별성이 같은 기준의 비교로 증명됨
- 자원·절차·일정·위험 대응·완료 기준이 있음
- 직접·확장·장기 효과가 구분되고 측정 기준이 있음
- 근거 뒤에 의미 해석과 소결론이 있음
- 같은 개념·수치·기간·주체가 일관됨
- 결론에 새로운 주장이나 자료가 없음
- 문단 첫 문장만 읽어도 전체 흐름이 이해됨
- 공식 평가항목별 판단과 근거 위치를 찾을 수 있음
- 원문에 없는 사실을 만들지 않음

## 출력 원칙

사용자가 요청한 결과를 가장 먼저 제공하라.

- 설계: 7부 구조 설계안을 먼저 제공하라.
- 초안: 제목과 7부 제출 원고를 먼저 제공하라.
- 진단: 가장 치명적인 구조·논리 결함부터 제시하라.
- 재작성: 7부 구조로 완성한 수정본을 먼저 제공하라.
- 최종 검수: 잔여 결함, 교체문, 제출 판정을 순서대로 제공하라.

근거 없는 예상 점수, 수상 확률, 수상 보장을 만들지 마라.

## 완료 기준

다음이 모두 성립할 때만 완료하라.

- 공식 제출조건과 평가항목을 확인함
- 핵심 결론이 한 문장으로 고정됨
- 원인과 해결 방안이 직접 대응함
- 원고가 유일한 7부 구조를 따름
- 모든 문단이 `주장 → 설명 → 근거·사례 → 소결론`을 따름
- 차별성과 실현 가능성이 구체적임
- 기대 효과와 측정 기준이 연결됨
- 문장이 짧고 구체적이며 용어가 일관됨
- 결론이 새 내용 없이 선택 이유를 묶음
- 사실을 발명하지 않음
