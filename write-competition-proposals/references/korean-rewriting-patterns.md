# 공격적 한국어 공모전 문장 패턴

## 1. 기본 리듬

공모전 문장은 `결론을 먼저 말하고 이유를 증명하는 리듬`으로 쓴다.

`판정 → 근거 → 논거 → 설계 → 효과 → 선택 결론`

한 문장에는 중심 명제 하나를 둔다. 문단 첫 문장은 답을 선언하고, 마지막 문장은 심사자가 내려야 할 판단으로 닫는다.

## 2. 약한 문장을 판정문으로 바꾸기

| 약한 표현 | 강한 구조 |
| --- | --- |
| `~하고자 한다` | `~한다` |
| `~할 수 있을 것으로 기대된다` | `~을 목표로 실행한다` / `~을 측정한다` |
| `도움이 될 수 있다` | `~의 [구체 단계]를 바꾼다` |
| `긍정적 효과가 있다` | `[지표]를 [측정법]으로 개선한다` |
| `검토가 필요하다` | `[주체]가 [시점]까지 [결정]한다` |
| `연계가 가능하다` | `[주체 A]가 [정보]를 [주체 B]에게 전달한다` |
| `추진될 예정이다` | `[주체]가 [단계]를 실행한다` |
| `의미가 있다` | `[평가항목]에서 [판정]을 만든다` |

확정한 설계와 계획은 단정하라. 검증 전 성과는 목표와 측정 문장으로 구체화하라.

## 3. 추상명사를 행동으로 바꾸기

`활성화`, `고도화`, `효율화`, `체계화`, `강화`, `확대`, `제고`, `촉진`, `연계`가 나오면 다음을 복원하라.

1. 누가
2. 무엇을
3. 어떤 절차로
4. 언제
5. 어떻게 바꾸는가

약함:

> 취약계층 지원 강화를 위한 통합관리체계를 구축한다.

강한 구조:

> `[운영주체]`는 `[분산된 정보·업무]`를 `[하나의 절차]`로 통합해 `[누락·지연]`을 차단한다.

## 4. 숨은 주체를 앞으로 끌어내기

피동형을 책임 문장으로 바꿔라.

- `정보가 제공된다` → `[주체]가 [대상]에게 [정보]를 제공한다`
- `사업이 추진된다` → `[주체]가 [기간]에 [사업 단계]를 실행한다`
- `관리가 이루어진다` → `[담당자]가 [기준]으로 [대상]을 관리한다`
- `연계가 가능하다` → `[주체 A]와 [주체 B]가 [정보·업무]를 연결한다`

실행주체가 여럿이면 역할을 표로 분리하고 책임자를 명시하라.

## 5. 목적–수단–결과를 분리하기

`~을 위한`, `~을 통한`, `~에 기반한`이 연속되면 세 문장으로 분리하라.

약함:

> 데이터 기반 맞춤형 지원을 통한 청년 취업역량 강화를 위한 플랫폼을 구축한다.

강한 구조:

> `[주체]`는 `[데이터]`로 청년의 `[구체 상태]`를 구분한다. 플랫폼은 상태별로 `[지원 행동]`을 제공한다. 이 구조는 `[직접 지표]`를 바꾸고 `[성과지표]`로 결과를 측정한다.

## 6. 인과를 문장에 드러내기

접속어가 아니라 실제 메커니즘을 써라.

약함:

> 접근성이 낮다. 따라서 플랫폼이 필요하다.

강한 구조:

> 정보가 기관별로 흩어져 사용자는 탐색을 반복한다. 본 제안은 기관별 정보를 한 경로에 연결해 탐색 단계를 줄인다. 단계 감소는 `[접근 시간·완료율]`의 변화로 측정한다.

문제와 기능 사이에 `원인`, 기능과 효과 사이에 `중간 변화`가 있어야 한다.

## 7. 근거를 논거로 밀어붙이기

수치나 사례 뒤에 반드시 의미와 설계 선택을 붙여라.

> `[기관·시점·대상]` 자료에서 `[사실]`이 확인됐다. 이는 `[핵심 원인·공백]`이 문제의 병목임을 보여준다. 따라서 본 제안은 `[그 병목을 바꾸는 개입]`을 핵심으로 채택한다.

`자료가 있다 → 필요하다`에서 끝내지 마라. 왜 그 자료가 이 설계를 선택하게 만드는지 써라.

## 8. 비교우위를 판정문으로 쓰기

약함:

> 기존 서비스보다 혁신적이고 편리하다.

강한 구조:

> 기존 `[대안]`은 `[단계]`에서 `[구체 공백]`을 남긴다. 본 제안은 `[메커니즘]`으로 그 단계를 `[새 절차]`로 전환한다. 그 결과 `[비교 기준]`에서 본 제안이 공모 목적을 더 직접적으로 달성한다.

비교 기준은 2~4개로 좁혀라. 형용사가 아니라 절차·시간·비용·대상 적합성·측정 가능성으로 승부하라.

## 9. 반론을 설계로 제압하기

반론을 소개하고 물러서지 마라. 선택을 막는 질문과 답을 같은 문단에 둔다.

> `[심사 의문]`은 `[구체 장치]`로 해소된다. `[주체]`는 `[단계]`에서 `[대체 절차·완료 기준]`을 적용한다. 따라서 `[평가항목]`에서 실행 가능성을 확보한다.

`그럴 수 있다`, `고려해야 한다`, `향후 검토한다`로 끝내지 마라.

## 10. 문단 첫 문장

| 문단 기능 | 첫 문장 공식 |
| --- | --- |
| 채택 결론 | `본 제안은 [문제]를 [메커니즘]으로 바꾸는 가장 직접적인 선택이다.` |
| 문제 | `[대상]은 [상황]에서 [손실]을 반복해서 겪는다.` |
| 원인 | `이 손실은 [구조적 원인] 때문에 반복된다.` |
| 기존 공백 | `현재 [대안]은 [결정적 단계]를 해결하지 못한다.` |
| 해법 | `본 제안은 [주체]가 [개입]하도록 절차를 전환한다.` |
| 차별성 | `본 제안의 승부처는 [같은 기준에서의 구체 차이]다.` |
| 실행 | `[주체]는 [기간] 동안 [단계]를 실행해 [완료 기준]을 달성한다.` |
| 효과 | `[중간 변화]는 [성과지표]의 변화로 이어진다.` |
| 결론 | `따라서 본 제안은 [공모 목적]을 [차별적 방식]으로 실현하는 선택이다.` |

자료가 뒷받침하는 범위에서 단어를 채워라.

## 11. 제목과 첫 세 문장

### 제목

`[대상·상황]의 [핵심 문제]를 [차별적 메커니즘]으로 바꾸는 [제안 유형]`

수단명만 나열하지 마라. `AI 기반 혁신 플랫폼`보다 누구의 어떤 문제를 무엇으로 바꾸는지가 먼저다.

### 첫 세 문장

1. 대상과 핵심 손실을 선언한다.
2. 구조적 원인과 기존 방식의 공백을 지목한다.
3. 제안의 개입·메커니즘·성과를 한 번에 제시한다.

첫 세 문장 안에 채택 이유가 없으면 서두를 다시 써라.

## 12. 기능별 문단 패턴

- 문제: `판정 → 규모·시급성 근거 → 대상의 손실 → 지금 해결해야 할 결론`
- 원인: `구조적 원인 → 반복 과정 → 기존 대응의 실패 지점 → 개입 레버`
- 제안: `핵심 개입 → 행위자·행동 → 정보·업무 흐름 → 직접 산출`
- 메커니즘: `개입 → 사용자·시스템 반응 → 중간 변화 → 성과지표`
- 차별성: `동일 비교 기준 → 기존 공백 → 제안의 차이 → 선택 판정`
- 실행: `주체 → 자원 → 단계 → 일정 → 완료 기준`
- 효과: `직접 산출 → 중간 변화 → 성과 → 측정 방법·시점`
- 결론: `공모 목적 → 결정적 우위 → 실행 의지 → 채택 결론`

## 13. 삭제 대상

- 현대사회는 빠르게 변화하고 있다.
- 그 중요성은 아무리 강조해도 지나치지 않다.
- 본 제안에서는 문제를 해결하고자 한다.
- 긍정적인 효과가 있을 것으로 기대된다.
- 향후 적극적으로 검토할 예정이다.
- 일정 부분 도움이 될 수 있다.
- 다양한 측면에서 의미가 있다.
- 다만 아직 단정하기 어렵다.

삭제한 자리는 실제 명제·근거·논거·실행·측정으로 채워라.

## 14. 최종 문장 감사

1. 첫 문장이 답을 선언하는가
2. 행위자가 앞에 있는가
3. 추상명사가 실제 행동으로 바뀌었는가
4. 문제–원인–개입–중간 변화–성과가 연결되는가
5. 근거 뒤에 논거와 설계 판단이 있는가
6. 비교가 같은 기준에서 이루어지는가
7. 계획에 담당·시점·완료 기준이 있는가
8. 문단 마지막이 심사 판단으로 닫히는가
9. 관성적 완곡어와 자기반박이 제거됐는가
10. 문단 첫 문장만 읽어도 채택 논리가 완성되는가
11. 원문에 없는 사실을 추가하지 않았는가
