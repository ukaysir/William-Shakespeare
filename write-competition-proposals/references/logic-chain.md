# 공모전 표준 논리 연결법

## 목차

1. 목적과 표준 논리표
2. 문제와 원인 구분
3. 원인–해결 일대일 대응
4. 해결 방안의 작동 확인
5. 효과 연결
6. 차별성 비교
7. 실현 가능성 확인
8. 정방향·역방향 검사
9. 논리 단절 탐지
10. 확정 판정

## 목적

논리 연결은 독자가 제안의 결론에서 문제·원인·해결·실행·효과를 거쳐 최종 선택 이유까지 헤매지 않고 따라가게 만드는 작업이다. 글의 방향은 다음 하나로 고정하라.

`제안 요약 → 문제 상황 → 문제의 원인 → 해결 방안 → 차별성과 실현 가능성 → 기대 효과 → 결론`

## 표준 논리표

초안 전에 다음 표를 작성하라. 표의 순서를 원고의 7부 구조와 같게 유지하라.

| 섹션 | 주장 | 설명 | 근거·사례 | 소결론 | 공식 평가항목 |
| --- | --- | --- | --- | --- | --- |
| 1. 제안 요약 | 무엇을 제안하는가 | 현재 상황과 핵심 문제를 왜 이 해결책으로 묶는가 | 핵심 사실 또는 확정 설계 | 글 전체의 실행 방향 |  |
| 2. 문제 상황 | 누가 어떤 손실을 겪는가 | 그 손실이 왜 중요하고 지속되는가 | 통계·현장 사례·공식 자료 | 해결해야 할 핵심 문제 |  |
| 3. 문제의 원인 | 문제가 왜 반복되는가 | 현상 뒤의 구조적·실질적 원인은 무엇인가 | 과정 자료·연구·업무 흐름·기존 대응 사례 | 해결책이 바꿔야 할 지점 |  |
| 4. 해결 방안 | 누가 무엇을 어떻게 하는가 | 해결 방안이 원인을 어떻게 바꾸는가 | 설계·절차·유사 사례 | 실행해야 할 핵심 방안 |  |
| 5. 차별성과 실현 가능성 | 왜 이 방안이 다르고 가능한가 | 기존 방식과의 차이와 실행 조건은 무엇인가 | 동일 기준 비교·역할·자원·일정·위험 대응 | 현실적인 선택이라는 판단 |  |
| 6. 기대 효과 | 무엇이 어떻게 달라지는가 | 직접 효과가 확장·장기 효과로 어떻게 이어지는가 | 기준선·목표·측정법·유사 결과 | 확인할 변화와 평가 기준 |  |
| 7. 결론 | 왜 지금 선택해야 하는가 | 필요성·원인 적합성·실행 가능성·효과가 어떻게 결합되는가 | 앞선 섹션에서 확인한 핵심 근거 | 최종 선택 이유 |  |

표의 `주장 → 설명 → 근거·사례 → 소결론`은 실제 문단의 유일한 작성 순서다.

## 문제와 원인 구분

다음 세 요소를 분리하라.

- 현황: 현재 어떤 일이 벌어지고 있는가
- 문제: 그 현상으로 누가 어떤 손실을 겪는가
- 원인: 그 손실이 왜 반복되는가

문제를 원인처럼 반복하지 마라. 원인은 절차, 정보, 자원, 책임, 접근 방식, 참여 유인처럼 해결 방안으로 바꿀 수 있는 수준까지 내려가라.

| ---: | --- | --- | --- | --- |
| 1 |  |  |  |  |

원인과 해결 방안의 번호와 용어를 일치시켜라. 원인에 대응하지 않는 기능은 삭제하거나 부가기능으로 분류하라. 해결되지 않은 원인이 남으면 범위를 줄이거나 방안을 추가하라.

## 해결 방안의 작동 확인

해결 방안마다 다음 질문에 답하라.

1. 누가 실행하는가
2. 누구를 대상으로 하는가
3. 무엇을 하는가
4. 어떤 절차로 운영하는가
5. 원인의 어느 지점을 바꾸는가
6. 실행 직후 무엇이 달라지는가

기능 목록에서 효과로 건너뛰지 마라. 실행주체의 행동과 대상의 변화를 구체적으로 써라.

## 효과 연결

6부에서는 다음 순서로 효과를 검증하라.

`해결 방안 실행 → 직접 효과 → 확장 효과 → 장기 효과 → 측정 기준`

- 직접 효과: 해결 방안이 바로 만드는 변화
- 확장 효과: 직접 효과가 행동·협업·시간·비용·접근성에 만드는 후속 변화
- 장기 효과: 확장 효과가 유지될 때 가능한 변화
- 측정 기준: 지표, 기준선, 목표, 측정법, 측정 시점

각 화살표가 왜 이어지는지 설명하라. 장기 효과는 목표 또는 검증 대상이라고 표시하고 달성 사실로 쓰지 마라.

## 실현 가능성 확인

다음 요소를 하나의 실행 설계로 결합하라.

`실행주체 → 자원 → 단계별 절차 → 일정 → 위험과 보완책 → 완료 기준`

실행 의지나 협력 예정이라는 표현을 실행 증거로 사용하지 마라. 확보되지 않은 협약·예산·승인을 사실로 쓰지 마라.

## 정방향 검사

다음 질문에 순서대로 답하라.

1. 대상은 어떤 손실을 겪는가
2. 그 손실은 어떤 원인 때문에 반복되는가
3. 해결 방안은 그 원인의 어느 지점을 바꾸는가
4. 누가 어떤 자원과 절차로 실행하는가
5. 실행 직후 어떤 직접 효과가 생기는가
6. 직접 효과는 어떤 확장·장기 효과로 이어지는가
7. 효과를 무엇으로 측정하는가
8. 그래서 왜 이 제안을 선택해야 하는가

답이 비는 지점에 실제 내용이나 근거를 보강하라. 접속어만 추가하지 마라.

## 역방향 검사

목표 효과에서 거꾸로 추적하라.

`장기 효과에 필요한 확장 효과 → 확장 효과에 필요한 직접 효과 → 직접 효과를 만들 해결 방안 → 해결 방안을 실행할 주체·자원·절차 → 해결해야 할 원인`

역추적에서 새 요소가 나오면 7부 구조의 해당 섹션에 같은 용어로 반영하라.

## 논리 단절 탐지

- 현황과 문제를 같은 문장으로 반복함
- 문제를 원인처럼 다시 말함
- 원인 분석 뒤에 무관한 기능이 등장함
- 해결 방안에 실행주체나 운영 절차가 없음
- 기존 방식과 본 제안을 다른 기준으로 비교함
- 실행 의지를 실현 가능성의 증거로 사용함
- 실행 활동을 기대 효과로 부름
- 직접 효과 없이 장기 효과로 점프함
- 통계가 문제만 보여주는데 해결책의 효과까지 주장함
- 결론에서 앞에 없던 가치·기능·자료를 추가함

단절을 발견하면 문장을 세게 바꾸지 말고 빠진 원인, 설명, 실행 조건, 직접 효과 또는 측정 기준을 채워라.

## 확정 판정

- 핵심 결론이 하나다.
- 7부 구조의 모든 섹션에 주장·설명·근거·사례·소결론이 있다.
- 문제와 원인이 구분된다.
- 모든 핵심 원인에 해결 방안이 대응한다.
- 해결 방안의 실행주체·대상·행동·운영방식이 보인다.
- 차별성이 같은 기준에서 증명된다.
- 실현 가능성에 자원·절차·일정·위험 대응·완료 기준이 있다.
- 직접·확장·장기 효과와 측정 기준이 연결된다.
- 모든 공식 평가항목이 7부 구조 안에서 회수된다.
- 결론이 새 정보 없이 선택 이유를 확정한다.
