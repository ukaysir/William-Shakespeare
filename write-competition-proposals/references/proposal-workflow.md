## 운영 원칙

다음 우선순위를 지켜라.

1. 자격·제출조건·사실 무결성
2. 공식 평가항목과 배점
3. 문제–원인–개입–결과의 논리
4. 심사자의 정보 탐색 편의
5. 문장 수준의 명료성과 완성도

좋은 문장을 만들기 전에 무엇을 증명해야 하는지 확정하라. 공식 양식의 필드와 순서를 임의의 서사 구조로 대체하지 마라. 양식 안에서 논리를 강화하라.

공고문마다 평가체계가 다르므로 `창의성·실현가능성·효과성`을 보편적인 고정 배점으로 취급하지 마라. 공식 공고에 없는 평가항목은 보조 진단으로만 사용하라.

## 입력과 출처 잠금

작업을 시작할 때 다음 자료의 존재 여부와 판본을 확인하라.

| 우선순위 | 자료 | 확인할 항목 |
| --- | --- | --- |
| 1 | 공식 공고문 | 주최기관, 공모명, 공고일, 마감, 참가 자격, 분야 |
| 2 | 평가표·심사기준 | 항목명, 세부 질문, 배점, 과락, 동점 규칙 |
| 3 | 제출양식 | 필수 필드, 글자·페이지 제한, 첨부 방식, 서명 여부 |
| 4 | 사용자 자료 | 아이디어, 초안, 조사자료, 확정 수치, 유지할 표현 |
| 5 | 외부 근거 | 발행기관, 발행일, 조사대상, 단위, 원문 위치 |

파일명이 같아도 내용이 다를 수 있으므로 최신 판본과 수정 공지를 확인하라. 웹페이지, 첨부 공고, 제출양식이 충돌하면 임의로 선택하지 말고 충돌을 보고하라.

자료를 다음 상태로 구분하라.

- `확정`: 공식 자료 또는 사용자가 확정한 정보
- `잠정`: 합리적이지만 검증이 남은 정보
- `부재`: 필요한 정보가 제공되지 않음
- `충돌`: 둘 이상의 자료가 서로 다름

## 작업 모드 선택

사용자의 동사를 기준으로 가장 좁은 모드를 선택하라. 요청하지 않은 대규모 재작성을 수행하지 마라.

| 사용자 의도 | 모드 | 핵심 산출물 |
| --- | --- | --- |
| 아이디어를 구조화·기획 | 설계 | 제안 핵심, 평가 지도, 목차, 자료 공백 |
| 처음부터 원고 작성 | 초안 | 근거에 한정한 완성 원고와 미확정 표시 |
| 문제점만 파악 | 진단 | 우선순위 결함, 원문 근거, 수정 방향 |
| 문장·구조 개선 | 재작성 | 사실 보존 수정본과 주요 변경 설명 |
| 제출 직전 확인 | 최종 검수 | 제출 위험, 평가 대응, 일관성, 잔여 공백 |

여러 모드가 결합되면 `진단 → 사용자 범위 내 재작성 → 최종 검수` 순서로 수행하라. 진단만 요청받으면 수정본을 본문 대신 짧은 예시로 제한하라.

## 1단계: 공고문 잠금

### 1.1 공모전 신분표 만들기

먼저 다음 항목을 한 표로 잠가라.

| 필드 | 값 | 출처 | 상태 |
| --- | --- | --- | --- |
| 공모명 |  |  |  |
| 주최·주관 |  |  |  |
| 공모 목적 |  |  |  |
| 지원 분야 |  |  |  |
| 참가 자격 |  |  |  |
| 마감과 시간대 |  |  |  |
| 제출 파일·경로 |  |  |  |
| 분량·서식 |  |  |  |
| 필수 첨부 |  |  |  |
| 제외·실격 조건 |  |  |  |

자격 또는 실격 가능성이 있으면 글쓰기보다 먼저 `P0 / COMPLIANCE_RISK`로 보고하라.

최신 공식 연장·정정 공지를 확인한 뒤 마감일·시각·시간대를 현재 날짜와 비교하라. 마감일 당일인데 종료 시각이나 시간대가 없으면 임의로 종료 처리하지 말고 확인 필요로 남겨라. 연장 없이 마감이 지났으면 이번 회차는 `제출 불가`이며, 원고 개선은 `향후 회차 재사용` 작업으로 분리하라.

### 1.2 평가표 원형 보존하기

항목명을 요약하면서 원래 의미를 바꾸지 마라. 다음 필드를 추출하라.

| 평가항목 | 배점 | 공식 세부기준 | 심사자가 답해야 할 질문 | 과락·동점 영향 |
| --- | ---: | --- | --- | --- |
|  |  |  |  |  |

배점 합계를 계산해 누락 여부를 확인하라. 합계가 100이 아니어도 오류로 단정하지 말고 단계별 심사, 가산점 또는 정성평가 여부를 확인하라.

### 1.3 양식 우선순위 정하기

제출양식의 필드 순서는 유지하되, 각 필드 안에서는 결론을 먼저 배치하라. 같은 내용을 여러 필드에 요구하면 반복 복사하지 말고 필드 목적에 맞게 관점을 달리하라.

예:

- `필요성`: 누구에게 어떤 손실이 왜 발생하는가
- `제안내용`: 누가 무엇을 어떤 순서로 바꾸는가
- `기대효과`: 어떤 지표가 어떤 경로로 변하는가
- `실현가능성`: 인력·예산·일정·권한·위험을 어떻게 감당하는가

## 2단계: 제안 핵심 잠금

원고를 쓰기 전에 다음 열 칸을 채워라. 정보가 없으면 추측하지 말고 공백 표시를 사용하라.

| 요소 | 확인 질문 |
| --- | --- |
| 대상 | 정확히 누가 문제를 겪는가 |
| 상황 | 언제·어디서 문제가 나타나는가 |
| 문제 | 관찰 가능한 불편·비용·위험은 무엇인가 |
| 원인 | 기존 방식이 왜 해결하지 못하는가 |
| 개입 | 누가 무엇을 어떻게 바꾸는가 |
| 작동원리 | 개입이 어떤 중간 변화를 만드는가 |
| 결과 | 대상에게 무엇이 달라지는가 |
| 측정 | 어떤 지표·기준선·시점으로 확인하는가 |
| 실행 | 인력·예산·일정·협력·권한은 무엇인가 |
| 차별성 | 비교대안보다 어떤 상황에서 더 적합한가 |

### 핵심문 만들기

다음 구조를 사용하되 자연스러운 한국어로 다듬어라.

> `[대상]`이 `[상황]`에서 겪는 `[문제]`를, `[실행주체]`가 `[개입]`하여 `[작동원리]`를 만들고 `[검증 가능한 결과 또는 목표]`로 이어지게 한다.

결과가 검증되지 않았으면 `만든다` 대신 `목표로 한다`, `검증한다`, `기대한다`처럼 증거 수준에 맞는 동사를 써라.

### 30초 재구성 기준

제목과 첫 문단만 별도로 떼어 다음 네 질문에 답하라.

1. 누구의 어떤 문제인가
2. 무엇을 제안하는가
3. 어떻게 작동하는가
4. 어떤 변화가 예상되는가

원문에 없는 내용을 보충해야 답할 수 있다면 통과로 판정하지 마라.

## 3단계: 평가항목 역설계

각 평가항목을 `주장–근거–위치` 단위로 역설계하라.

| 평가항목 | 배점 | 심사 질문 | 답해야 할 핵심 주장 | 필요한 근거 | 현재 근거 | 배치 위치 | 상태 |
| --- | ---: | --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |  | 충족/부분/부재/충돌 |

다음 규칙을 적용하라.

- 높은 배점과 과락 항목을 먼저 보강하라.
- 한 근거가 여러 항목을 지원해도 각 항목에서 의미를 다르게 설명하라.
- 평가항목명을 제목에 기계적으로 반복하지 말고, 해당 항목을 판단할 사실을 제공하라.
- `창의성`은 새롭다는 선언이 아니라 비교대안과 차이를 보여라.
- `실현가능성`은 의지 표현이 아니라 자원·권한·일정·위험 대응을 보여라.
- `효과성·파급효과`는 긍정적이라는 선언이 아니라 수혜대상, 변화 경로, 지표를 보여라.
- `구체성·내용충실성`은 분량이 아니라 빠진 의사결정 정보가 없는지를 보여라.

평가표가 없으면 공모 목적과 제출양식에서 암묵적 질문을 추출하되 `공식 평가항목`이라고 부르지 마라. 일반 보조 기준으로 표시하라.

## 4단계: 근거와 정보 공백 분리

모든 핵심 주장을 다음 네 유형 중 하나로 분류하라.

- 자료가 충분한 주장
- 자료는 있으나 주장과 연결되지 않은 주장
- 표현만 있고 근거가 없는 주장
- 필요한 주장 자체가 빠진 항목

근거 처리의 세부 규칙은 [evidence-integrity.md](evidence-integrity.md)를 읽고 적용하라.

자료가 없을 때는 다음 표시를 사용하라.

- `[근거 필요: 무엇을 확인해야 하는지]`
- `[수치 확인 필요: 기준선·단위·시점]`
- `[실행주체 미정]`
- `[비교대안 조사 필요]`
- `[공고 확인 필요: 충돌 내용]`

빈칸을 숨기기 위해 추상적인 문장을 늘리지 마라. 정보 공백 목록을 원고와 분리해 사용자가 다음 조사 행동을 알 수 있게 하라.

## 5단계: 목차와 문단 설계

### 5.1 공식 양식이 있을 때

양식의 제목과 순서를 보존하라. 각 필드에 다음을 배정하라.

1. 필드가 답해야 할 심사 질문
2. 첫 문장의 결론
3. 필요한 근거
4. 다음 필드로 넘길 질문
5. 분량 예산

### 5.2 자유 형식일 때

기본 흐름을 다음처럼 시작하되 공식 평가표에 맞춰 조정하라.

1. 제안 요약: 문제–대상–해법–효과
2. 문제와 필요성: 현상–규모–원인–기존 공백
3. 제안 내용: 행위자–행동–절차–사용 장면
4. 차별성: 비교대안–차이–적합 조건
5. 실행계획: 단계–담당–자원–일정–위험
6. 기대효과와 측정: 변화 경로–지표–기준선–목표–시점
7. 확산·지속가능성: 재원–운영–확장 조건

### 5.3 문단 질문–답변 사슬 만들기

각 문단에 다음 표식을 임시로 붙여라.

| 문단 | 독자 질문 | 첫 문장의 답 | 근거 | 이 문단이 새로 더하는 정보 | 다음 질문 |
| --- | --- | --- | --- | --- | --- |
|  |  |  |  |  |  |

문단이 새 정보를 추가하지 않으면 삭제·병합·역할 변경을 검토하라. 다만 독자의 방향을 알려주는 전환문은 `탐색 기능`으로 인정하라.

## 6단계: 초안 또는 재작성

다음 순서로 작성하라.

1. 평가항목별 핵심 주장부터 써라.
2. 문단 첫 문장에 답을 배치하라.
3. 근거를 붙이고 그 근거가 주장에 주는 의미를 설명하라.
4. 행위자·행동·조건·결과를 드러내라.
5. 예외, 제약, 위험 대응을 필요한 위치에 넣어 신뢰를 높여라.
6. 제목과 첫 문단을 마지막에 다시 작성하라.

한국어 문장 변환은 [korean-rewriting-patterns.md](korean-rewriting-patterns.md)를 읽고 적용하라. 14개 글쓰기 기준의 판정은 [writing-rubric.md](writing-rubric.md)를 읽고 적용하라.

재작성할 때는 다음 경계를 지켜라.

- 허용: 순서 변경, 문장 분리·병합, 주어 복원, 중복 삭제, 용어 통일, 주장 강도 낮추기
- 조건부 허용: 명시적 가정 추가, 사용자가 제공한 자료의 재해석
- 금지: 새 통계·성과·기관 협의·사용자 반응·법적 가능성을 사실처럼 추가하기

## 7단계: 제출 전 검수

다음 순서로 검사하라.

### 게이트 A — 제출 가능성

- 참가 자격과 분야가 맞는가
- 마감, 파일 형식, 분량, 서명, 첨부가 맞는가
- 표절·중복 수상·지식재산권·제외조건 위험이 있는가

### 게이트 B — 평가 가능성

- 모든 공식 평가항목에 근거 또는 명시적 공백이 있는가
- 높은 배점과 과락 항목의 근거가 앞쪽에 있는가
- 심사자가 근거 위치를 빠르게 찾을 수 있는가

### 게이트 C — 논리와 근거

- 문제가 원인과 연결되는가
- 개입이 원인을 실제로 건드리는가
- 결과가 작동원리에서 이어지는가
- 통계·사례·비교가 주장에 연결되는가
- 사실·추정·목표가 구분되는가

### 게이트 D — 표현과 일관성

- 제목과 첫 문단이 제안 핵심을 식별시키는가
- 문단마다 하나의 중심 답이 있는가
- 대상·용어·수치·단위·기간이 일치하는가
- 같은 결함을 여러 항목에서 반복 보고하지 않았는가

최종 출력 형식은 [output-contracts.md](output-contracts.md)를 읽고 사용자의 요청 모드에 맞게 선택하라.

## 정보가 부족할 때

안전하게 진행할 수 있으면 멈추지 말고 다음처럼 범위를 제한하라.

- 공고문 없음: 일반 보조 기준으로만 진단하고 공식 점수 예측을 하지 마라.
- 평가표 없음: 공모 목적·양식에서 심사 질문 후보를 추출하고 `추정`으로 표시하라.
- 근거 없음: 구조와 질문 목록을 만들고 결과 수치는 쓰지 마라.
- 초안 일부만 있음: 제공된 범위만 진단하고 문서 전체에 대한 통과 판정을 하지 마라.
- 사용자 의도 불명확: 가장 보존적인 모드인 `진단`을 기본으로 삼아라.

## 중단하고 확인할 조건

다음 중 하나가 결과를 실질적으로 바꾸면 사용자에게 확인하라.

- 서로 다른 공식 판본이 참가 자격·마감·평가표에서 충돌함
- 어느 공모 분야에 지원할지에 따라 제안 구조가 달라짐
- 사용자가 제공하지 않은 민감정보나 비공개 데이터가 필요함
- 사실 추가 없는 재작성으로는 요청한 효과 수치를 제시할 수 없음
- 원문의 핵심 아이디어를 바꾸어야만 평가항목을 충족할 수 있음
- 법률·의료·재무 판단을 사실로 확정해야 함

확인 전에도 수행할 수 있는 공고 정리, 공백 표시, 원문 진단은 계속하라.
