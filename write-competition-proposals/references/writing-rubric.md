## 판정 원칙

문체 취향이 아니라 심사자의 관찰 가능한 과업으로 판정하라. `좋다`, `매끄럽다`, `설득력 있다`만으로 평가하지 마라.

각 판정에 다음 네 요소를 붙여라.

1. 원문 위치 또는 짧은 인용
2. 독자가 실패하는 구체적 과업
3. 평가 또는 이해에 미치는 영향
4. 최소 수정 방향

원문에 없는 사실을 머릿속으로 보충한 뒤 통과시키지 마라. 심사자가 추론해야 하는 연결은 문서에 없는 연결로 취급하라.

한 결함을 여러 기준에서 반복 보고하지 마라. 가장 근본적인 기준 하나에 배치하고 `연관 영향`만 덧붙여라.

공식 평가표가 이 루브릭보다 우선한다. 이 루브릭은 공식 점수를 대신하지 않고 공식 항목의 근거가 전달되는지를 검사한다.

## 점수와 심각도

### 충족도

- `0 — 부재·모순`: 필요한 정보가 없거나 서로 충돌해 독자가 올바르게 답할 수 없음
- `1 — 부분 충족`: 정보는 있으나 늦게 나오거나 모호하고, 독자가 연결·해석해야 함
- `2 — 명확히 충족`: 필요한 정보가 명시적이고 일관되며 해당 위치에서 바로 찾을 수 있음
- `N/A — 비적용`: 문서 유형상 정말 적용되지 않을 때만 사용하고 이유를 적음

숫자만 제시하지 말고 근거를 붙여라. 14개 점수를 단순 합산해 수상 확률이나 예상 심사점수로 바꾸지 마라.

### 심각도

- `P0`: 자격·사실성·제안 정체·핵심 이해를 무너뜨림
- `P1`: 공식 평가항목의 점수를 직접 약화함
- `P2`: 탐색 속도·가독성·완성도를 낮춤

충족도와 심각도는 독립적으로 매겨라. 예를 들어 문서 일관성이 `0`이어도 사소한 띄어쓰기 차이면 `P2`, 대상 인원 수가 충돌하면 `P0` 또는 `P1`이 될 수 있다.

## 1. 초기 식별성

기존의 `후킹`을 클릭 유도나 자극성이 아니라 제안의 정체를 즉시 밝히는 능력으로 판정하라.

### 검사

제목과 첫 문단만 떼어 다음 네 요소를 적어라.

- 문제: 어떤 손실·불편·위험인가
- 대상: 누가 겪는가
- 제안: 무엇을 바꾸는가
- 변화: 어떤 결과 또는 목표로 이어지는가

### 판정

- `0`: 두 요소 이상이 없거나 제안이 구호 수준임
- `1`: 네 요소가 흩어져 있거나 핵심 개입이 모호함
- `2`: 독자가 원문 표현만으로 네 요소를 정확히 재구성함

### 실패 신호

- 제목이 `AI 기반 혁신 플랫폼 구축`처럼 수단만 말함
- 첫 문단이 배경 설명으로 끝나고 제안은 뒤에서 등장함
- `문제를 해결하고자 한다`고만 쓰고 누가 무엇을 하는지 없음
- 과장된 형용사가 제안의 작동 방식을 대신함

### 수정

첫 문단의 첫 1~2문장 안에 대상–문제–핵심 개입–변화를 배치하라. 제목에 모든 정보를 억지로 넣지 말고, 제목과 첫 문단이 함께 네 요소를 완성하게 하라.

## 2. 명료성

처음 보는 독자가 한 번 읽고 제안을 자기 말로 정확히 설명할 수 있는지를 판정하라.

### 검사

원문의 전문용어를 그대로 반복하지 말고 다음을 평이한 말로 재진술하라.

1. 현재 누가 무엇 때문에 어려운가
2. 제안자는 무엇을 바꾸려 하는가
3. 바뀐 뒤 사용자는 무엇을 다르게 하는가
4. 성공 여부를 어떻게 알 수 있는가

재진술에 원문 밖의 가정이 들어갔는지 표시하라. 에이전트의 재진술은 대리 검사일 뿐 실제 독자 검사의 증거라고 주장하지 마라.

### 판정

- `0`: 서로 다른 두 설명이 가능하거나 핵심 질문에 답할 수 없음
- `1`: 대체로 이해되나 용어·지시어·생략 때문에 한 번 이상 되읽어야 함
- `2`: 주요 요소가 한 가지 의미로 재진술되고 미정 정보도 명확히 표시됨

### 실패 신호

- 정의하지 않은 약어와 내부 용어
- `이를`, `해당`, `관련`, `효율화`의 지시 대상이 불명확함
- 사용자, 운영기관, 시스템의 역할이 섞임
- 같은 단어가 문단마다 다른 뜻으로 쓰임

### 수정

전문용어를 없애기보다 첫 사용에서 뜻과 역할을 정의하라. 모호한 지시어를 실제 대상명으로 바꾸고, 한 문장 안의 역할 변화를 분리하라.

## 3. 논리 흐름

각 문단이 앞 문단에서 자연스럽게 생긴 질문에 답하는지를 판정하라.

### 검사

문단마다 다음을 한 줄로 적어라.

`독자 질문 → 문단의 첫 답 → 근거 → 의미 → 다음 질문`

다음 핵심 연결도 별도로 확인하라.

`문제 → 원인 → 개입 → 중간 변화 → 최종 결과 → 측정`

### 판정

- `0`: 핵심 연결이 끊기거나 개입이 제시한 원인을 건드리지 않음
- `1`: 순서는 대체로 맞지만 연결 근거를 독자가 추론해야 함
- `2`: 각 문단의 역할과 인과·추론 연결이 명시적임

### 실패 신호

- 문제 규모 다음에 원인 설명 없이 기능 목록이 나옴
- 효과가 개입의 작동원리 없이 바로 선언됨
- 실행계획이 제안 내용보다 먼저 나와 무엇을 실행하는지 불명확함
- 결론이 본문에서 제시하지 않은 주장을 추가함

### 수정

전환어를 추가하는 데 그치지 말고 빠진 논리 단계를 보충하라. 정보가 없으면 `[원인 근거 필요]`처럼 공백을 드러내라.

## 4. 정보 전진성

같은 주장을 표현만 바꾸어 반복하지 않고, 문단과 문장이 새 판단 정보를 추가하는지를 판정하라.

### 검사

각 문장에 다음 기능 중 하나를 붙여라.

- `주장`
- `근거`
- `해석`
- `작동원리`
- `비교`
- `실행`
- `제약·위험`
- `측정`
- `탐색 안내`

인접 문장의 의미를 한 문장으로 바꾸었을 때 동일하면 중복 후보로 표시하라.

### 판정

- `0`: 핵심 분량이 반복·구호로 채워지고 새 정보가 거의 없음
- `1`: 일부 중복이 있으나 문단별 역할은 구분됨
- `2`: 필요한 요약 반복을 제외하면 각 단위가 새 판단 정보를 더함

### 실패 신호

- `중요하다`, `필요하다`, `시급하다`를 근거 없이 반복함
- 제안 요약과 기대효과가 같은 문장을 되풀이함
- 여러 기능 설명이 이름만 다르고 사용자 행동은 같음
- 결론이 본문 전체를 길게 재서술함

### 수정

반복 문장 중 가장 구체적인 문장을 남기고 나머지를 근거·비교·측정 정보로 교체하라. 요약 반복은 위치와 목적이 분명할 때만 허용하라.

## 5. 구체성

추상명사보다 행위자·행동·대상·조건·결과가 드러나는지를 판정하라.

### 검사

핵심 문장에서 다음 다섯 칸을 채워라.

| 행위자 | 행동 | 대상·도구 | 조건·시점 | 직접 결과 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

두 칸 이상 비어 있으면 추상 문장으로 의심하라.

### 판정

- `0`: `활성화`, `고도화`, `연계 강화`가 실제 행동을 대신함
- `1`: 행동은 있으나 담당자·조건·결과 중 일부가 모호함
- `2`: 독자가 실제 수행 장면과 책임 주체를 재현할 수 있음

### 실패 신호

- `통합관리체계를 구축한다`에서 누가 무엇을 통합하는지 없음
- `인식 개선을 추진한다`에서 대상·방법·변화 지표가 없음
- `맞춤형 서비스를 제공한다`에서 무엇을 기준으로 어떻게 달라지는지 없음

### 수정

명사를 지우는 것보다 숨은 동사를 복원하라. `분석을 실시한다`를 `담당자가 월별 신청 데이터를 분석한다`처럼 바꾸되 원문에 없는 주체는 확정하지 말고 표시하라.

## 6. 문장 기능성

기존의 `정보밀도`를 한 문장에 정보를 많이 넣는 정도가 아니라 평가에 필요한 기능을 수행하는 정도로 판정하라.

### 검사

각 문장에 하나 이상의 기능 태그를 붙이고 다음을 확인하라.

- 이 문장이 없으면 심사자의 어떤 판단이 어려워지는가
- 앞 문장과 다른 정보를 제공하는가
- 다음 문장을 이해하는 데 필요한가

### 판정

- `0`: 장식·구호·관습적 서두가 많아 삭제해도 의미가 거의 같음
- `1`: 기능은 있으나 중복되거나 더 짧게 전달할 수 있음
- `2`: 각 문장이 주장·근거·해석·탐색 중 필요한 역할을 수행함

### 실패 신호

- `현대사회는 빠르게 변화하고 있다` 같은 범용 서두
- `큰 의미가 있을 것으로 사료된다` 같은 비정보성 결론
- 평가에 쓰이지 않는 연혁·배경이 핵심보다 김
- 형용사만 늘고 판단 정보가 늘지 않음

### 수정

삭제만 고집하지 마라. 문장의 목적이 필요하면 실제 주장·근거·해석으로 바꾸고, 문단 방향을 알려주는 짧은 탐색 문장은 유지하라.

## 7. 인지부하

독자가 한 번에 유지하고 연결해야 하는 개념이 과도하지 않은지를 판정하라.

### 검사

다음 중 두 가지 이상이면 분리 후보로 표시하라.

- 중심 주장이 둘 이상임
- 행위자 또는 시점이 중간에 바뀜
- 조건·예외·목적이 세 겹 이상 중첩됨
- 긴 관형절 뒤에 핵심 명사가 늦게 나옴
- 괄호와 약어가 연속됨
- 하나의 열거가 서로 다른 분류 기준을 섞음

한국어 문장에 영어의 고정 단어 수 기준을 기계적으로 적용하지 마라. 길이보다 명제 수와 구조를 보라.

### 판정

- `0`: 되읽지 않으면 주어–서술어와 논리 관계를 복원하기 어려움
- `1`: 의미는 명확하지만 한 문장에 여러 판단이 몰림
- `2`: 문장마다 중심 명제가 하나이고 연결 관계가 쉽게 보임

### 실패 신호

- 문제·원인·해법·효과가 한 문장에 모두 들어감
- `~하며`, `~하고`, `~함으로써`가 연속됨
- 정의와 예외가 본문보다 긴 괄호 안에 들어감

### 수정

주제 변화, 행위자 변화, 사실과 해석의 경계에서 문장을 나눠라. 나눈 뒤 인과관계가 사라지지 않도록 접속 표현이나 주어를 복원하라.

## 8. 강조 구조

중요한 내용이 문서 앞쪽과 문단 첫 문장에 배치되는지를 판정하라.

### 검사

- 제목과 첫 문단에 제안 핵심이 있는가
- 각 문단 첫 문장만 이어 읽어도 전체 논리가 보이는가
- 높은 배점·과락 항목의 핵심 근거가 쉽게 보이는가
- 표·목록·굵은 글씨가 중요도와 일치하는가

### 판정

- `0`: 핵심 결론이 후반에 숨어 있고 앞부분은 배경뿐임
- `1`: 결론은 있으나 문단 중간 또는 긴 수식 뒤에 나타남
- `2`: 결론 우선 구조가 유지되고 강조 수단이 우선순위를 반영함

### 실패 신호

- 문제 배경을 한 페이지 설명한 뒤 마지막에 제안이 등장함
- 첫 문장이 `이에 본 제안에서는`으로 시작하지만 실제 답은 뒤에 있음
- 모든 문장을 굵게 표시해 강조 차이가 사라짐

### 수정

문단 첫 문장을 요약이 아니라 해당 문단의 판단 가능한 결론으로 바꿔라. 근거는 결론 뒤에 배치하되 공식 양식 순서는 보존하라.

## 9. 근거 통합

통계·사례·비교자료가 장식이 아니라 주장과 연결되는지를 판정하라.

### 검사

각 핵심 근거에 다음 사슬을 작성하라.

`주장 → 근거 → 근거의 의미 → 제안에 주는 함의`

근거의 출처·시점·대상·단위·한계는 [evidence-integrity.md](evidence-integrity.md)에 따라 내부 검수하라. 모든 한계를 근거 문장 뒤에 반복하지 말고, 의미를 바꾸는 범위 정보만 가까이 두며 나머지는 별도 확인 목록이나 원고 말미에서 회수하라.

### 판정

- `0`: 출처 없는 수치, 맥락 없는 통계, 근거보다 강한 결론이 있음
- `1`: 출처와 수치는 있으나 주장과의 연결이 약하거나 필요한 범위 정보를 회수하기 어려움
- `2`: 근거가 특정 주장을 지지하고 의미가 명확하며, 필요한 범위·한계를 해당 근거 가까이 또는 별도 섹션에서 회수할 수 있음

### 실패 신호

- 통계 뒤에 `따라서 필요하다`만 있고 어떤 설계 판단인지 없음
- 전국 통계로 특정 지역의 규모를 단정함
- 상관관계 자료로 제안의 인과효과를 확정함
- 목표 수치를 이미 달성된 성과처럼 씀
- 근거 문장마다 한계와 유보를 반복해 핵심 의미와 설계 판단이 묻힘

### 수정

근거 문장 다음에 `이 수치는 무엇을 보여주며 제안의 어떤 선택을 정당화하는가`를 한 문장으로 써라. 근거가 약하면 주장 문장 자체의 범위와 동사를 조정하거나 추가 조사 항목으로 돌려라. 근거 카드의 한계를 원고에 그대로 복사하지 마라.

## 10. 작동 시나리오

기존의 `장면화`를 감성적 서사가 아니라 서비스·정책·사업의 실제 동작을 재현하는 능력으로 판정하라.

### 검사

대표 사용 흐름에서 다음 여섯 요소를 채워라.

1. 촉발 상황
2. 사용자의 첫 행동
3. 실행주체 또는 시스템의 반응
4. 다음 행동과 정보 이동
5. 예외·실패 시 처리
6. 사용자가 체감하는 직접 결과

### 판정

- `0`: 기능명만 있고 사용 흐름을 재현할 수 없음
- `1`: 정상 흐름은 보이나 주체·순서·예외 중 일부가 빠짐
- `2`: 대표 흐름과 최소 한 가지 예외 대응을 구체적으로 그릴 수 있음

### 실패 신호

- `AI가 맞춤 정보를 제공한다`에서 입력·판단 기준·출력이 없음
- 기관 간 연계를 말하지만 누가 어떤 정보를 언제 넘기는지 없음
- 이상적인 성공 장면만 있고 미응답·오류·거절 처리가 없음

### 수정

한 명의 대표 사용자를 과도하게 극화하지 말고 행동과 시스템 반응을 시간순으로 써라. 검증되지 않은 화면·알림·자동화 기능은 사실처럼 추가하지 마라.

## 11. 기억성

독자가 제안의 차별적 메커니즘을 한 문장으로 회상할 수 있는지를 판정하라.

### 검사

문서를 덮었다고 가정하고 다음 문장을 완성하라.

> 이 제안은 `[대상]`의 `[문제]`를 `[차별적 개입 또는 작동원리]`로 바꾸는 제안이다.

문서에 없는 표현을 만들어야 완성되거나 다른 제안에도 그대로 적용되면 기억성이 약하다.

### 판정

- `0`: `플랫폼`, `활성화`, `혁신` 같은 범용어만 남음
- `1`: 대상과 문제는 기억되나 개입의 차이가 흐림
- `2`: 문제–대상–차별적 작동원리가 짧고 정확하게 회상됨

### 실패 신호

- 브랜드명만 독특하고 기능은 설명되지 않음
- 기능이 너무 많아 무엇이 핵심인지 알 수 없음
- 제목, 요약, 본문이 서로 다른 핵심 표현을 사용함

### 수정

하나의 핵심 메커니즘을 선택하고 제목·요약·문단 첫 문장에서 같은 표준 용어로 반복하라. 부가기능은 핵심과 구분하라.

## 12. 주장 강도와 어조

자신감과 과장, 전문성과 난해함의 균형을 사실 상태에 맞춰 판정하라.

### 검사

핵심 문장을 다음 상태로 분류하라.

- 확인된 사실
- 자료에 근거한 해석
- 추정·예측
- 계획·목표
- 가정·미확정

동사가 상태보다 강하지 않은지 확인하라.

### 판정

- `0`: 근거 없는 확정·최상급·유일성 또는 지나친 비전문적 표현이 있음
- `1`: 전반적으로 적절하지만 일부 예측과 사실이 섞임
- `2`: 사실 상태에 맞는 범위와 동사로 직접 표현하고, 필요한 한계는 정해진 위치에서 회수할 수 있음

### 실패 신호

- `완벽히 해결한다`, `유일하다`, `획기적이다`를 비교 근거 없이 사용함
- 모든 문장을 `~할 수 있을 것으로 기대된다`로 약화함
- 전문용어를 권위의 신호처럼 과도하게 사용함
- 계획을 이미 협의·확정된 것처럼 표현함
- 주장 뒤마다 `다만`, `아직`, `단정할 수 없다`를 붙여 앞의 결론을 스스로 회수함

### 수정

근거가 충분하면 수동적 완곡어를 줄이고 직접 말하라. 근거가 부족하면 형용사를 늘리거나 자기반박 문장을 덧붙이지 말고 주장 문장 자체의 범위를 줄여 `목표로 한다`, `검증한다`, `가정한다`처럼 상태에 맞는 동사를 써라. 나머지 한계와 확인 사항은 지정된 별도 위치에 모아라.

## 13. 심사 편의성

심사자가 평가항목별 점수를 줄 근거를 빠르게 찾고 인용할 수 있는지를 판정하라.

### 검사

평가항목 하나를 임의로 골라 다음을 측정하라.

- 항목에 대한 핵심 답을 찾을 수 있는가
- 답의 근거 위치가 명확한가
- 근거를 다른 항목과 혼동하지 않는가
- 표·제목·문단 첫 문장이 탐색을 돕는가

### 판정

- `0`: 근거가 흩어져 있거나 평가항목과 연결할 수 없음
- `1`: 찾을 수 있으나 여러 문단을 조합해야 함
- `2`: 항목별 핵심 답과 근거를 짧은 탐색으로 찾고 인용할 수 있음

### 실패 신호

- 평가표의 핵심 항목이 별도 위치 없이 여러 섹션에 흩어짐
- 표 제목과 열 이름이 무엇을 증명하는지 말하지 않음
- 핵심 수치의 기준선·목표·시점이 다른 페이지에 따로 있음
- 공식 양식의 질문과 답의 초점이 다름

### 수정

평가항목별 근거 지도를 먼저 만들고 원고 위치를 조정하라. 문단 첫 문장과 표 제목을 심사 질문에 대한 답으로 바꿔라.

## 14. 문서 일관성

동일한 대상·용어·수치·단위·기간·역할이 문서 끝까지 유지되는지를 판정하라.

### 검사

다음 원장을 만들고 모든 출현 위치를 대조하라.

| 유형 | 표준값 | 변형·충돌 | 위치 | 조치 |
| --- | --- | --- | --- | --- |
| 제안명 |  |  |  |  |
| 대상 |  |  |  |  |
| 핵심 용어 |  |  |  |  |
| 수치·단위 |  |  |  |  |
| 기간·단계 |  |  |  |  |
| 실행주체 |  |  |  |  |

### 판정

- `0`: 핵심 사실이 충돌하거나 용어 변화가 의미를 바꿈
- `1`: 의미는 같지만 표기와 범위가 흔들림
- `2`: 표준값이 유지되고 필요한 차이는 정의됨

### 실패 신호

- `청년`, `취업준비생`, `지역 청년`을 같은 대상으로 혼용함
- 본문과 표의 대상 인원·예산·기간이 다름
- 월간·연간 수치를 단위 표시 없이 비교함
- 운영기관과 협력기관의 책임이 문단마다 바뀜

### 수정

표준 용어와 수치를 먼저 확정한 뒤 전체 위치를 일괄 점검하라. 서로 다른 범위를 의도했다면 이름을 다르게 정의하라. 어떤 값이 맞는지 근거가 없으면 임의로 통일하지 말고 충돌로 남겨라.

## 기준 충돌 해결

다음 우선순위로 충돌을 해결하라.

1. 사실 정확성과 제출조건
2. 공식 평가항목에 필요한 정보
3. 논리 연결과 주장 강도
4. 심사 탐색 편의
5. 문장 간결성과 문체

### 정보 전진성 대 강조 구조

제안 핵심을 제목·요약·결론에서 반복할 수 있다. 각 반복의 기능을 `식별`, `판단`, `회상`으로 다르게 하고 같은 문단 안의 동의어 반복은 줄여라.

### 문장 기능성 대 인지부하

한 문장에 정보를 더 넣어 밀도를 높이지 마라. 필요한 정보를 여러 짧은 명제로 나누되 연결을 명확히 하라.

### 초기 식별성 대 어조

강한 형용사로 주목을 끌지 마라. 구체적인 대상·문제·개입·변화를 앞에 두어 식별성을 높여라.

### 사실 무결성 대 주장 선명성

한계를 삭제해 주장을 사실보다 강하게 만들지도, 모든 주장 뒤에 한계를 붙여 결론을 약하게 만들지도 마라. 내부 검수 기록은 완전하게 유지하고, 제출 원고에서는 사실 상태에 맞는 범위와 동사를 사용한 뒤 한계·전제·미확정 사항을 별도 위치에 모아라. 단, 분리하면 해당 문장이 거짓이 되거나 중대한 오해를 만드는 정보는 가까이 표시하라.

### 작동 시나리오 대 분량 제한

대표 정상 흐름 하나와 점수에 중요한 예외 하나만 선택하라. 모든 사용자 여정을 서사화하지 마라.

### 전문성 대 명료성

정확성에 필요한 전문용어는 유지하고 첫 사용에서 짧게 정의하라. 쉬운 말로 바꾸어 의미가 흐려지면 원어와 설명을 함께 사용하라.

필요한 기준만 다음 표로 보고하라. 모든 항목이 문제가 없을 때 14개를 길게 설명하지 마라.

| --- | ---: | --- | --- | --- | --- |
|  | 0/1/2 | P0/P1/P2 |  |  |  |

2. 점수에 가장 큰 영향을 주는 한두 결함이 무엇인지
3. 어떤 순서로 고치면 되는지
