# 공모전 공격적 설득 루브릭

## 판정 원칙

문체 취향이 아니라 `심사자가 이 제안을 선택할 충분한 이유를 문서에서 즉시 회수할 수 있는가`로 판정하라. 모든 진단은 원문 위치, 실패한 심사 판단, 점수 영향, 교체 행동을 붙여라.

공식 평가표가 이 루브릭보다 우선한다. 이 루브릭은 공식 항목에서 최고점 판단을 만들어내는 논리와 표현을 검사한다.

## 판정 등급

- `0 — 붕괴`: 핵심 판단을 만들 수 없거나 논리가 충돌함
- `1 — 약함`: 정보는 있으나 심사자가 연결·해석해야 함
- `2 — 장악`: 판정 문장과 근거가 명시되어 즉시 점수화할 수 있음

## 수정 우선순위

- `P0`: 자격·사실·제안 정체·핵심 논리를 무너뜨림
- `P1`: 공식 평가항목의 득점을 직접 깎음
- `P2`: 속도·가독성·기억성을 낮춤

## 1. 채택명제 단일성

### 검사

원고 전체가 증명하는 결론을 `심사위원회는 이 제안을 선택해야 한다. 왜냐하면 ...` 형식의 한 문장으로 복원하라.

### 판정

- `0`: 핵심 제안이 둘 이상이거나 선택 이유를 한 문장으로 만들 수 없음
- `1`: 제안은 하나지만 결론과 이유가 여러 섹션에서 흔들림
- `2`: 하나의 채택명제가 제목·요약·본문·결론을 지배함

### 수정

부가 기능을 핵심 메커니즘 아래 통합하고, 모든 섹션을 채택명제의 한 이유로 재배치하라.

## 2. 초기 장악력

### 검사

제목과 첫 세 문장에서 다음을 복원하라.

`대상 → 문제 → 원인 또는 기존 공백 → 개입 → 핵심 성과`

### 판정

- `0`: 배경·유행·수단명만 있고 제안 정체가 없음
- `1`: 요소는 있으나 뒤늦게 나오거나 흩어짐
- `2`: 첫 세 문장만으로 제안과 선택 이유가 보임

### 수정

첫 문장을 문제 판정, 둘째 문장을 원인·공백, 셋째 문장을 개입·메커니즘·성과로 다시 써라.

## 3. 공모 목적 직결성

### 검사

공모 목적의 핵심 동사와 제안의 최종 성과가 실제 인과로 연결되는지 확인하라.

### 판정

- `0`: 공모 주제의 단어만 반복함
- `1`: 방향은 맞지만 어떤 성과로 목적을 달성하는지 흐림
- `2`: 제안 성과가 공모 목적의 구체적 달성 경로로 제시됨

### 수정

`제안 성과 → 공모 목적에 주는 직접 효과 → 심사 결론`을 한 문단으로 닫아라.

## 4. 문제의 규모·시급성

### 검사

누가 어떤 상황에서 어떤 손실을 겪고, 왜 지금 바꿔야 하는지 확인하라.

### 판정

- `0`: 중요하다는 선언만 있음
- `1`: 현상은 보이지만 대상·규모·시급성 중 하나가 약함
- `2`: 대상·손실·규모·시점이 근거로 입증됨

### 수정

일반론을 제거하고 대상의 관찰 가능한 시간·비용·기회·접근성 손실로 바꿔라.

## 5. 원인 타격력

### 검사

`문제 → 구조적 원인 → 개입 레버`를 채우고 제안이 원인을 직접 바꾸는지 확인하라.

### 판정

- `0`: 현상을 원인으로 반복하거나 기능이 원인과 무관함
- `1`: 원인과 개입이 연결되지만 중간 설명이 필요함
- `2`: 개입이 원인의 특정 단계·행동·정보 흐름을 직접 바꿈

### 수정

문제 표면이 아니라 반복을 만드는 절차·인센티브·정보·책임 구조를 지목하고 개입 지점을 맞춰라.

## 6. 작동 메커니즘

### 검사

다음 사슬을 원문 표현으로 채워라.

`개입 → 사용자·시스템 반응 → 직접 산출 → 중간 변화 → 성과지표`

### 판정

- `0`: 기능 목록에서 효과로 점프함
- `1`: 정상 흐름은 보이지만 중간 변화 또는 지표가 약함
- `2`: 각 단계가 행위자·행동·결과로 명시되고 측정까지 연결됨

### 수정

접속어를 늘리지 말고 빠진 중간 변화와 측정지표를 추가하라.

## 7. 주장–근거–논거 폐쇄성

### 검사

핵심 주장마다 다음을 채워라.

`주장 → 근거 → 왜 이 근거가 주장을 증명하는가 → 심사 판정`

### 판정

- `0`: 근거 없는 주장 또는 주장과 무관한 자료가 있음
- `1`: 근거는 있으나 의미를 심사자가 추론해야 함
- `2`: 근거의 의미와 평가 판단이 한 논증 단위로 닫힘

### 수정

근거 뒤에 `이 사실은 무엇을 보여주며 어떤 설계 선택을 정당화하는가`를 직접 써라.

## 8. 비교우위

### 검사

기존 방식과 제안을 같은 대상·상황·비용·시간·성공 기준에서 비교하라.

### 판정

- `0`: `혁신적`, `차별적` 같은 형용사뿐임
- `1`: 차이는 있으나 비교 기준 또는 선택 판정이 흐림
- `2`: 결정적 공백과 제안의 우위가 같은 기준에서 증명됨

### 수정

승부처를 1~3개로 좁히고 `기존 공백 → 제안의 메커니즘 → 심사 결론`으로 써라.

## 9. 실행 증명력

### 검사

`주체 → 자원 → 단계 → 일정 → 완료 기준`을 채워라.

### 판정

- `0`: 의지와 전망만 있음
- `1`: 일정·역할은 있으나 자원 또는 완료 기준이 약함
- `2`: 누가 무엇으로 언제까지 어떻게 끝낼지 증명됨

### 수정

추상적 추진 계획을 단계별 책임과 통과 기준으로 바꿔라.

## 10. 성과·측정력

### 검사

활동·산출·성과를 구분하고 성과마다 기준선·목표·측정법·시점을 확인하라.

### 판정

- `0`: 활동이나 산출을 성과로 포장함
- `1`: 성과는 있으나 지표·측정법·시점 중 하나가 약함
- `2`: 변화 경로와 성과지표가 실행계획에 결합됨

### 수정

사용자의 행동·시간·비용·접근성 변화로 성과를 재정의하고 측정 설계를 붙여라.

## 11. 평가항목 회수력

### 검사

각 공식 항목에 대해 심사자가 인용할 판정 문장과 근거 위치를 찾게 하라.

### 판정

- `0`: 공식 항목 하나 이상에 답이 없음
- `1`: 답은 있으나 여러 위치를 조합해야 함
- `2`: 항목별 판정 문장·근거·원고 위치가 즉시 회수됨

### 수정

평가항목별 근거 지도를 만들고 높은 배점의 판정 문장을 앞에 배치하라.

## 12. 주장 강도

### 검사

확인 사실·확정 설계·실행 계획이 직접 표현됐는지, 빈 과장과 완곡어가 남았는지 확인하라.

### 판정

- `0`: 근거 없는 최상급 또는 관성적 유보가 결론을 무너뜨림
- `1`: 대체로 직접적이나 일부 문장이 흐리거나 비어 있음
- `2`: 모든 핵심 문장이 증거 범위의 최대 강도로 표현됨

### 수정

`~하고자 한다`, `기대된다`, `도움이 될 수 있다`를 행위·성과·측정 동사로 교체하라.

## 13. 문단 전진성

### 검사

각 문장에 `판정·근거·논거·메커니즘·비교·실행·측정·전환` 중 하나의 기능을 부여하라.

### 판정

- `0`: 구호와 반복이 핵심 분량을 차지함
- `1`: 문단 역할은 있으나 중복이 남음
- `2`: 모든 문단이 논리사슬의 새 고리나 득점 근거를 추가함

### 수정

가장 구체적인 문장을 남기고 반복 문장을 새 근거·논거·비교·측정으로 교체하라.

## 14. 문서 일관성

### 검사

제안명, 대상, 문제, 원인, 개입, 수치, 기간, 주체, 성과지표의 표준값을 문서 전체에서 대조하라.

### 판정

- `0`: 핵심 값이나 역할이 충돌함
- `1`: 의미는 같지만 범위와 용어가 흔들림
- `2`: 표준값이 제목·요약·본문·표·결론에서 유지됨

### 수정

표준값을 먼저 확정하고 모든 출현 위치를 같은 의미로 통일하라.

## 15. 최종 채택 압력

### 검사

마지막 문단이 다음을 모두 회수하는지 확인하라.

`공모 목적 → 문제의 시급성 → 결정적 메커니즘 → 비교우위 → 실행 의지 → 측정 가능한 성과 → 채택 결론`

### 판정

- `0`: 단순 요약이나 전망으로 끝남
- `1`: 핵심 가치는 보이지만 선택 이유 하나 이상이 빠짐
- `2`: 심사자가 왜 지금 이 제안을 선택해야 하는지 단호하게 확정됨

### 수정

새 정보를 추가하지 말고 본문에서 입증한 승부처만 압축해 마지막 문장을 채택 판정으로 끝내라.

## 종합 판정 형식

문제가 있는 기준만 보고하라.

| 우선순위 | 기준 | 점수 | 원문 위치 | 실패한 심사 판단 | 즉시 교체 행동 |
| --- | --- | ---: | --- | --- | --- |
| P0/P1/P2 |  | 0/1/2 |  |  |  |

종합 결론은 세 문장으로 끝낸다.

1. 현재 원고가 심사자에게 강요하는 핵심 결론
2. 그 결론을 무너뜨리는 가장 큰 논리 단절 또는 득점 손실
3. 가장 먼저 교체할 구조나 문장
