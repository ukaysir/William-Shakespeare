## 목적

근거를 자료 목록으로 남기지 말고 문단의 중심 주장과 연결하라. 모든 근거는 유일한 문단 공식인 `주장 → 설명 → 근거·사례 → 소결론` 안에서 사용하라. 소결론에서 근거의 의미를 해석하고 다음 판단으로 연결하라.

## 1. 문장 상태와 표현 강도

| 상태 | 의미 | 적합한 표현 |
| --- | --- | --- |
| 확인 사실 | 공식 자료·원자료로 확인됨 | `확인됐다`, `나타났다`, `차지한다` |
| 사실 해석 | 확인 사실에서 직접 도출한 의미 | `보여준다`, `의미한다` |
| 확정 설계 | 제안자가 채택한 기능·절차 | `제공한다`, `연결한다`, `전환한다` |
| 실행 계획 | 앞으로 수행할 행동 | `실행한다`, `운영한다`, `측정한다` |
| 목표 | 달성하려는 기준 | `목표로 한다`, `달성 기준으로 삼는다` |
| 추정 | 계산·모형에 따른 예상 | `추정한다`, `산출한다` |

확인 사실, 설계, 계획, 목표, 추정을 서로 바꾸지 마라. 목표를 이미 달성한 성과처럼 쓰지 마라.

## 2. 근거의 역할

근거마다 주된 역할 하나를 부여하고 7부 구조의 해당 위치에 배치하라.

| 역할 | 증명할 내용 | 원고 위치 | 적합한 근거 |
| --- | --- | --- | --- |
| 문제 근거 | 대상의 손실과 시급성 | 2. 문제 상황 | 공식 통계, 현장 기록, 원자료 |
| 원인 근거 | 문제가 반복되는 이유 | 3. 문제의 원인 | 과정 데이터, 연구, 업무 흐름 |
| 해결 근거 | 해결 방안이 원인을 바꾸는 방식 | 4. 해결 방안 | 유사 사례, 연구, 시제품·설계 자료 |
| 비교 근거 | 기존 방식과 다른 구체적 지점 | 5. 차별성과 실현 가능성 | 동일 기준 비교, 절차·시간·비용 자료 |
| 실행 근거 | 실제 수행할 능력과 조건 | 5. 차별성과 실현 가능성 | 역할, 자원, 일정, 경험, 확보된 협력 |
| 효과 근거 | 어떤 변화를 어떻게 확인할지 | 6. 기대 효과 | 기준선, 목표, 지표, 측정법 |

문제 근거만으로 특정 해결 방안의 효과를 주장하지 마라.

## 3. 표준 근거 카드

핵심 문단마다 다음 카드를 내부 작업표에 작성하라.

| 문단 요소 | 내용 |
| --- | --- |
| 주장 | 이 문단이 증명할 한 가지 판단 |
| 설명 | 판단이 성립하는 이유·원인·작동 방식 |
| 근거·사례 | 출처가 직접 보여주는 사실 또는 확인된 설계·계획 |
| 소결론 | 근거의 의미와 해당 섹션에서 내려야 할 판단 |
| 출처 | 기관·자료명·시점·원문 위치 |
| 공식 평가항목 | 이 문단이 뒷받침하는 공식 항목 |

근거·사례 뒤의 소결론이 비면 자료 나열이다. 소결론에는 `이 사실이 무엇을 의미하며 왜 앞의 주장을 뒷받침하는가`를 써라.

## 4. 출처 우선순위

1. 최신 공식 공고·평가표·제출양식
2. 해당 기관의 공식 원자료·통계·법령·사업 문서
3. 원 연구논문·원 데이터·표준
4. 공신력 있는 기관의 분석 보고서
5. 신뢰할 수 있는 보도와 전문가 해설

핵심 수치는 검색 결과 요약에서 끝내지 말고 원문에서 대상·시점·단위를 확인하라. 웹 자료를 사용한 원고에는 제출양식에 맞춰 근거 가까이에 출처를 표시하라.

## 5. 수치 확인

수치마다 다음을 확인하라.

- 대상과 지역
- 조사·측정 시점
- 분자·분모
- 단위
- 비교 기준
- 실제값·목표값·추정값 구분
- 원문 위치

`20%에서 30%`는 `10%p 증가`이며 상대 증가율은 `50%`다. 기간·단위·대상이 다른 값을 직접 비교하지 마라.

수치를 넣은 문단은 다음 역할을 모두 수행해야 한다.

1. 주장: 수치가 뒷받침할 판단
2. 설명: 수치가 중요한 이유
3. 근거·사례: 기관·시점·대상·수치
4. 소결론: 수치의 의미와 문제·원인·해결 판단의 연결

## 6. 원인과 해결 방안의 근거 연결

원인별로 해결 방안이 직접 대응하는지 확인하라.

| 원인 | 원인 근거 | 해결 방안 | 해결 근거 | 직접 효과 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

유사 사례나 연구가 있으면 해결 방안이 작동하는 구체적 단계에 배치하라. 직접 근거가 없으면 효과를 단정하지 말고 실행 단계에서 검증할 지표를 설계하라.

## 7. 비교 근거

기존 방식과 본 제안을 같은 조건에서 비교하라.

- 같은 사용자와 상황
- 같은 기능 범위
- 같은 기간·비용·단위
- 같은 성공 기준

`최초`, `유일`, `최고`, `압도적`은 충분한 비교자료가 있을 때만 사용하라. 자료가 부족하면 확인 가능한 기능·절차 차이를 직접 써라.

## 8. 실행 근거

실현 가능성은 다음 요소로 확인하라.

`실행주체 + 보유·조달 자원 + 단계별 절차 + 일정 + 위험과 보완책 + 완료 기준`

`적극 추진한다`, `협력할 예정이다`, `충분히 가능하다`는 실행 근거가 아니다. 확보되지 않은 예산·공간·협력·승인을 사실로 쓰지 마라.

## 9. 기대 효과와 측정

| 효과 단계 | 변화 | 기준선 | 목표 | 측정법 | 시점 |
| --- | --- | ---: | ---: | --- | --- |
| 직접 효과 |  |  |  |  |  |
| 확장 효과 |  |  |  |  |  |
| 장기 효과 |  |  |  |  |  |

기준선이 없으면 목표 숫자를 발명하지 말고 첫 실행 단계에서 기준선을 측정하도록 설계하라. 정량지표가 부적합하면 관찰 가능한 판정 기준을 제시하라.

활동과 효과를 구분하라.

- 활동: 실행주체가 수행하는 행동
- 산출: 활동 직후 만들어지는 결과물
- 직접 효과: 대상의 즉각적인 변화
- 확장 효과: 직접 효과가 만드는 후속 변화
- 장기 효과: 변화가 유지·확장될 때의 결과

앱 출시, 교육 횟수, 참여 인원은 그 자체로 문제 해결 효과가 아니다.

## 10. 자료가 부족할 때

1. 공식 자료와 원자료를 추가 확인하라.
2. 확보한 근거가 직접 뒷받침하는 주장으로 범위를 줄여라.
3. 효과 숫자 대신 원인–해결 연결과 측정 설계를 구체화하라.
4. 확인되지 않은 우위 대신 확인 가능한 기능·절차 차이를 써라.
5. 필수값 하나가 전체 결과를 바꿀 때만 사용자에게 확인하라.

## 11. 금지

- 통계·인터뷰·수요·성과·협의·승인·예산·성능을 만들기
- 계획을 완료 사실로 바꾸기
- 목표를 달성 성과로 바꾸기
- 상관관계를 확정 인과로 바꾸기
- 비교 기준을 제안에 유리하게 바꾸기
- 원문의 수치·대상·기간을 조용히 변경하기
- 긴 출처 목록으로 근거 해석의 부재를 숨기기

## 12. 최종 근거 검사

- 모든 중심 주장에 설명과 근거·사례가 있는가
- 근거 뒤의 소결론이 의미를 해석하는가
- 문제·원인·해결·비교·실행·효과 근거가 구분되는가
- 수치의 대상·시점·단위가 일치하는가
- 원인과 해결 방안이 직접 대응하는가
- 비교가 같은 기준에서 수행되는가
- 실행 조건이 구체적인가
- 직접·확장·장기 효과와 측정 기준이 연결되는가
- 공식 평가항목에 필요한 근거가 7부 구조 안에 있는가
- 새 사실을 만들지 않았는가
