# 공모전 근거 공격력 관리

## 목적

근거를 장식이나 출처 목록으로 쓰지 마라. 모든 근거를 `어떤 주장을 왜 증명하며 어느 평가항목의 어떤 판정을 만드는가`에 연결하라. 사실을 발명하지 않으면서도 확보한 증거가 허용하는 최대 강도로 주장하라.

## 1. 주장 상태와 최대 강도

| 상태 | 의미 | 기본 동사 |
| --- | --- | --- |
| 확인 사실 | 공식 자료·원자료로 확인됨 | `확인됐다`, `나타났다`, `차지한다` |
| 근거 해석 | 사실에서 직접 도출한 의미 | `보여준다`, `입증한다`, `의미한다` |
| 확정 설계 | 제안자가 채택한 기능·절차 | `제공한다`, `연결한다`, `전환한다`, `차단한다` |
| 실행 계획 | 수행할 단계·행동 | `실행한다`, `운영한다`, `측정한다` |
| 목표 | 달성하려는 성과 기준 | `목표로 한다`, `달성 기준으로 삼는다` |
| 추정 | 계산·모형에 따른 예상 | `추정한다`, `산출한다` |

확인 사실·확정 설계·실행 계획을 `가능하다`, `기대된다`, `검토한다`로 약화하지 마라. 목표와 추정을 이미 달성한 사실로 바꾸지 마라.

## 2. 공모전 근거의 여섯 역할

근거마다 하나의 주된 역할을 부여하라.

| 역할 | 증명할 것 | 강한 근거 예 |
| --- | --- | --- |
| 문제 근거 | 문제가 실제로 크고 시급함 | 공식 통계, 원자료, 현장 기록 |
| 원인 근거 | 문제가 반복되는 구조적 이유 | 과정 데이터, 연구, 업무 흐름 |
| 해법 근거 | 개입이 원인을 바꾸는 메커니즘 | 유사 개입 결과, 실증연구, 시제품 데이터 |
| 비교 근거 | 기존 방식보다 선택할 이유 | 동일 기준 비교표, 절차·비용·시간 데이터 |
| 실행 근거 | 실제로 수행할 능력 | 역할·자원·일정, 실적, 확보된 협력 |
| 성과 근거 | 어떤 변화가 측정되는가 | 기준선, 중간지표, 목표, 측정법 |

문제 근거로 특정 해법의 효과를 증명하지 마라. 각 역할의 증거를 논리사슬의 해당 고리에 배치하라.

## 3. 주장–근거–논거–판정 카드

핵심 주장마다 다음 카드를 내부 작업표에 작성하라.

| 필드 | 내용 |
| --- | --- |
| 주장 | 심사자가 받아들여야 할 한 문장 |
| 근거 | 출처가 직접 보여주는 사실 |
| 논거 | 그 사실이 왜 주장을 증명하는가 |
| 설계 함의 | 그래서 제안은 무엇을 선택하는가 |
| 비교우위 | 같은 기준에서 기존 방식보다 무엇이 나은가 |
| 평가 판정 | 어느 공식 항목에서 어떤 결론을 만드는가 |
| 출처 | 기관·자료명·시점·원문 위치 |

카드가 `근거`에서 `판정`으로 바로 점프하면 논거가 빠진 것이다. 근거와 판단 사이의 원리를 한 문장으로 명시하라.

## 4. 출처 우선순위

1. 최신 공식 공고·평가표·제출양식
2. 해당 기관의 공식 원자료·통계·법령·사업 문서
3. 원 연구논문·원 데이터·표준
4. 공신력 있는 기관의 분석 보고서
5. 신뢰할 수 있는 보도와 전문가 해설

핵심 수치는 검색 결과 요약에서 끝내지 말고 원문을 열어 대상·시점·단위를 확인하라. 웹 자료를 사용한 원고에는 공모전 양식에 맞춰 출처를 가까이 표시하라.

## 5. 수치를 득점 근거로 전환

수치마다 다음을 잠가라.

- 대상과 지역
- 조사·측정 시점
- 분자·분모
- 단위
- 비교 기준
- 실제값·목표값·추정값 구분
- 원문 위치

수치를 제시한 뒤 반드시 설계 판단으로 연결하라.

> `[기관·시점·대상]` 자료에서 `[수치]`가 확인됐다. 이는 `[원인 또는 공백]`이 핵심 병목임을 보여준다. 따라서 본 제안은 `[그 병목을 직접 바꾸는 개입]`에 자원을 집중한다.

`20%에서 30%`는 `10%p 증가`, 상대 증가율은 `50%`다. 기간·단위·대상이 다른 값을 직접 비교하지 마라.

## 6. 인과 증명

효과를 주장하기 전에 다음 연결을 완성하라.

`개입 → 바뀌는 절차·행동 → 직접 산출 → 중간 변화 → 성과지표`

각 화살표마다 `왜 다음 단계가 발생하는가`를 한 문장으로 설명하라. 유사 사례나 연구가 있으면 그 연결에 배치하고, 없으면 실행 단계에서 직접 검증할 중간지표를 설계하라.

활동, 산출, 성과를 구분하라.

- 활동: 수행하는 행동
- 산출: 행동 직후 만들어지는 결과물
- 성과: 대상의 행동·시간·비용·접근성 변화
- 파급: 성과가 확장되어 만드는 장기 변화

앱 출시, 교육 횟수, 참여 인원은 그 자체로 문제 해결 성과가 아니다.

## 7. 비교우위 증명

비교 대상과 본 제안을 같은 조건에서 비교하라.

- 같은 사용자와 상황
- 같은 기능 범위
- 같은 기간·비용·단위
- 같은 성공 기준

`최초`, `유일`, `최고`, `압도적`은 충분한 비교자료가 있을 때만 쓴다. 자료가 부족하면 더 약한 형용사로 물러서지 말고, 확인 가능한 구체적 차이를 직접 써라.

> 기존 방식은 `[단계]`에서 `[공백]`을 남긴다. 본 제안은 `[메커니즘]`으로 그 단계를 `[구체 변화]`로 전환한다. 따라서 `[공모 상황]`에서는 본 제안이 `[평가 기준]`을 더 직접적으로 충족한다.

## 8. 실행 증명

실현가능성을 다음 식으로 증명하라.

`담당 주체 + 보유·조달 자원 + 단계별 행동 + 일정 + 완료 기준`

`적극 추진한다`, `협력할 예정이다`, `충분히 가능하다`는 실행 증거가 아니다. 각 단계의 담당과 완료 조건을 붙여라.

## 9. 성과 측정

| 성과지표 | 기준선 | 목표 | 측정 방법 | 측정 시점 | 평가항목 |
| --- | ---: | ---: | --- | --- | --- |
|  |  |  |  |  |  |

기준선이 없으면 근거 없이 목표 숫자를 만들지 말고 첫 단계에서 기준선을 측정하도록 설계하라. 정량지표가 적합하지 않으면 관찰 가능한 판정 기준을 제시하라.

## 10. 자료가 부족할 때의 공격적 처리

자료 부족을 제출문에 길게 설명하지 마라. 다음 순서로 해결하라.

1. 공식·원자료를 추가 조사한다.
2. 이미 확보한 근거가 직접 증명하는 명제로 바꾼다.
3. 효과 숫자 대신 작동 메커니즘과 측정 설계를 강화한다.
4. 확인되지 않은 비교우위 대신 확인 가능한 기능·절차 차이를 쓴다.
5. 필수값 하나가 전체 결과를 바꿀 때만 사용자에게 짧게 묻는다.

## 11. 금지

- 통계·인터뷰·수요·성과·협의·승인·예산·성능을 새로 만들기
- 계획을 완료 사실로 바꾸기
- 목표를 달성 성과로 바꾸기
- 상관관계를 확정 인과로 바꾸기
- 비교 기준을 제안에 유리하게 바꾸기
- 원문의 수치·대상·기간을 조용히 변경하기
- 긴 출처 목록으로 논거의 부재를 숨기기

## 12. 최종 근거 감사

- 모든 핵심 주장에 직접 근거가 있는가
- 근거와 주장 사이의 논거가 명시됐는가
- 문제·원인·해법·비교·실행·성과 근거가 구분됐는가
- 수치의 대상·시점·단위가 일치하는가
- 개입부터 성과까지 각 인과 연결이 설명됐는가
- 비교가 같은 기준에서 수행됐는가
- 실행이 주체·자원·일정·완료 기준으로 증명됐는가
- 각 근거가 공식 평가항목의 판정으로 회수됐는가
- 문장 강도가 증거가 허용하는 최대치인가
- 새 사실을 만들지 않았는가
